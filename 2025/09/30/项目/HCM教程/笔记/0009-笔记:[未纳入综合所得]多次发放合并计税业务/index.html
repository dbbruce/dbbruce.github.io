[未纳入综合所得]-如何处理多次发放合并计税业务
场景一：年终奖单独算税，且年终奖单独发放，和月度工资相互独立；理解为：年终奖与月工资不在同一个薪酬方案
  • 1.创建独立的年终奖薪酬方案，“计算个税”属性设置为否
  • 2.薪酬方案中创建以下薪酬项目
项目名称 设置说明 增项 计税项目 应发项 代扣代缴（补发补扣项） 成本项 定薪支付 周期性 减项
年终奖应发各类具体项目 需要设哪些薪酬项目视具体情况而定，可能只有一个年终奖应发，也可能是多个项目之和；计税项目选择“全年一次性奖金”的作用在于统计年度总收入，目前默认加和处理 增 全年一次性奖金 是 否 是 否 否 —
年终奖税金 因为全年一次性奖金计税和月度工资扣税方法不同，不可以使用系统“本次扣税”字段，需要单独创建年终奖税金项目，并设置为税后补扣项目，从实发中直接扣减 减 否 是 是 是 否 否 —
  • 3.创建薪酬公式
  • 4.理解该场景下系统预制项是否有用，以及计算逻辑
场景二：年终奖单独算税，但是年终奖和月度工资合并发放；理解为：年终奖与月工资在同一个薪酬方案中
  • 1.无需创建独立的年终奖薪酬方案，直接将年终奖相关项目创建在月度工资薪酬方案中
  • 2.薪酬方案中创建以下薪酬项目

年终奖实发 是否需要创建该项目需视情况而定 辅 否 否 否 否 否 否 —
