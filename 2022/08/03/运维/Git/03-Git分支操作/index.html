Git分支操作

1.常见的分支规范如下

MASTER分支(MASTER是主分支，是代码的核心
DEVELOP分支 (DEVELOP最新开发成果的分支)大功能
RELEASE分支 (为发布新产品设置的分支)
HOTFIX分支 (为了修复软件BUG缺陷的分支)
FEATURE分支 (为开发新功能设置的分支)小功能

2.查看位于那个分支

  • 查看当前分支
    git status 查看所有分支信息
  • 创建分支
  • 切换分支
  • 合并分支
    将hotfix修改的数据合并到master分支
    注意，合并前必须要先切换到master分支，然后再执行merge命令
  • 解决版本分支的冲突问题 手动解决冲突 再次提交，解决冲突 更新分支
