02-tomcat部署

2022年08月02日
运维 > Java生态

1.安装tomcat

简单来说，tomcat本身就是java写的一个脚本，所有不用安装

2.启动服务

启动后进程不叫tomcat，使用netstat要过滤java，ps最好过滤java

注意：8005，8080端口一定要启动，如果不能启动，稍等下，可能慢，tomcat需要内存里的大量数据

3.启动的特别慢

提示:如果检查端口时，8005端口启动非常慢，可用使用下面的命令用urandom替换random (非必须操作)

使用cat /dev/random,看下是否有大量随机数，有表示正常
cat /dev/random # 乱码
strings /dev/random # 好点，可以看的乱码

4.关闭

/usr/local/tomcat/bin/shutdown.sh

5.访问

http://192.168.1.12:8080

6.写一个简单测试页面

http://192.168.1.12:8080/test.jsp

7.修改server.xml配置文件

  • tomcat配置文件严格分隔大小写

  • 配置文件成对出现，必须注意结尾在哪里，如 <Host …..>,成对，注意结束标志

  • 连接端口

  • 虚拟主机

  • 默认打开的的虚拟主机

  • docBase设置首页,在Host中，新增Context，原文件没有，需要自己新增

    在webapps下新建一个base，添加一个index.jps文件

  • 跳转
    访问/abc跳转到/var/www/html
    www.a.com/abc 实际打开的是 /var/www/html文件
