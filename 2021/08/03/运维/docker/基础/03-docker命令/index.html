03-Docker命令

2021年08月03日
运维 > docker > 基础

docker命令

镜像命令

  • docker images //查看镜像列表

  • docker history 镜像名:版本 //查看镜像制作历史，主要看几层

  • docker inspect 镜像/容器/网络交换机 //查看镜像底层信息，后面跟镜像就查看镜像信息，后面跟容器就查看容器信息

    跟容器

  • docker pull 镜像:版本 //下载镜像

  • docker push [OPTIONS] NAME[:TAG] //上传镜像

  • docker rmi id/名称:版本 //删除本地镜像，删除镜像，必须删除使用过这个镜像的容器，容器运行和停止的都要先删除
    images id唯一时，可以使用id删除
    -f 强制删除

  • docker save //镜像另存为tar包
    docker save 名称:标签 -o xxx.tar

  • docker load //使用tar包导入镜像
    docker load -i 本地镜像包

  • docker search //搜索镜像
    name 镜像名称，下载时需要使用这个名称
    OFFICIAL 带有ok的是官网镜像，没有ok的就是用户自己做的

  • docker tag 原名称:版本 新名称：新版本 //修改镜像名称和标签，相当于创建一个软连接，既可以改名称，也可以改标签，或者全改
    修改的名称和标签，不能与别的镜像一样；
    修改后的image id，会相同

容器命令

  • docker ps -a // 查看容器进程信息，加 -a查看全部容器，包括停止和运行的；不加-a查看正在运行的docker
    -a 全部显示
    -q 简略显示，只显示id
    -aq 简略显示，显示全部id

CONTAINER ID 容器id
IMAGE 镜像
COMMAND 容器里执行的命令
CREATED 创建的时间
STATUS 状态
PORTS
NAMES 名字，不指定随机名称

  • docker run // 启动镜像，生成容器run
    -i 交互式 -it 打开一个交互式终端
    -t 终端
    -d 容器后台运行run
    -p(小写) 指定端口映射，格式：主机(宿主)端口:容器端口
    -P(大写) 随机端口映射，容器内部端口随机映射到主机的端口（49153起始 49153到65535）
    –name –name=centosrun 指定容器名称
    -v 指定容器和物理机的映射目录:docker run -itd -v /data1:/data2 -P cenginx:v1 : -v 物理机目录：容器目录 注意每运行一个容器，宿主机上就会多一个veth***开头的网卡，veth作用就是流量转发
  • docker stop 容器id // 停止容器
  • docker start 容器id // 启动容器
  • docker restart 容器id // 重新启动容器
  • docker attach 容器id //进入容器，attach退出容器，容器会被关闭；特殊情况使用；只能看交互的，非交互的看不了；
    attach 进入的进程id是1，也就是所有的父进程，这时退出的话，父进程id1退出，所有进程就会退出，容器就关闭了； 误进入想要保留容器，使用下面命令退出，保留容器；
  • docker exec -it 容器id 命令 //进入容器，退出，容器不会关闭，99%使用这个进入容器；
  • docker inspect 容器id //查看容器底层信息
  • docker top 容器id //查看容器进程列表，docker外部查看docker内部进程
  • docker rm //删除容器, docker运行的不能删除，必须停止 删除全部容器
  • docker commit 容器id 新镜像名:新镜像版本
    使用镜像启动容器，在该容器基础上修改
    另存为一个新镜像
  • docker build 从 Dockerfile 和上下文构建镜像
    -f 指定dockerfile路径 -t 指定构建的镜像名和tag
