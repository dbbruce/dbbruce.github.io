15-日志切分

2021年09月15日
运维 > nginx

手动切分

1. 重命名旧日志文件

注意：linux是用file id对文件进行失败，即使修改了文件名，file id没有变化仍向旧文件中写日志

2. user方式重启nginx，产生新日志文件

nginx服务不会重启，日志文件 file id 会变化
kill -USR1 $(cat /usr/local/nginx/logs/nginx.pid)

启动前
重启file id

备注:/usr/local/nginx/logs/nginx.pid文件中存放的是nginx的进程PID号

重启后

自动切分

每周5的03点03分自动执行脚本完成日志切割工作
