Linux设置本地yum源

关防火墙

系统光盘开机自动挂载

/dev/cdrom：是Linux系统中的光驱目录，只能以只读方式挂载。
mnt：镜像挂载点，可以自定义创建，这里以系统已有的mnt目录为挂载点。
/dev/cdrom /mnt：就是将光驱设备挂载到系统根分区的mnt目录下。
iso9660：这是光盘的文件格式，这个不可修改。
defaults和0 0：这个是操作权限，无需修改，默认即可。

验证配置是否生效，会自动挂载

配置文件生效会出现下字样

查看挂载是否正常

删除其他yum源

创建yum源文件 CentOS7.repo

[CentOS7] # 这是文件id，必须和文件名一致，区分大小写；
name=CentOS7Yum # 这个是描述性信息，可以随意写，建议规范；
baseurl=file:///mnt # mnt表示光盘挂载点，用该光驱里的文件作为我的yum；
enabled=1 # 1表示启动，0表示停用；
gpgcheck=0 # 0表示不效验，由于是光驱是本地的，所以这里不校验

59-SELinux

2021年09月29日
运维 > linux云计算基础教程

SELinux

概念

  • Security-Enhanced Linux
    一套强化Linux安全的扩展模块
    美国国家安全局主导开发
  • SELinux的运作机制
    集成到Linux内核( 2.6及以上)
    操作系统提供可定制的策略、管理工具

查看selinux状态：sestatus

配置文件:/etc/selinux/config

重置安全上下文，也就是开启selinux的步骤，只重启并不生效，需要创建autorelable文件

  • 修改配置文件，开启selinux
  • 根下创建.autorelable文件, 注意不要缺失了点，创建的是隐藏文件，重启后这个文件会自动消失；
  • 重启

SELinux开关控制

  • 非Disabled状态下才可可用
  • 临时调整，setenforce 命令
  • 设为 1，对应强制模式
  • 设为 0，对应宽松模式

查看安全上下文(安全上下文，就是安全策略), 使用大写Z

  • Security Context，安全上下文为文件
    /目录/设备标记访问控制属性
  • 属性构成，比较关注的是访问类型
    用户:角色:访问类型:选项.
  • 查看文件的安全上下文
  • 查看目录的安全上下文
  • 查看进程的安全上下文
  • 修改安全上下文，一般操作规律
    移动的文件，原有的上下文属性不变 创建或复制的文件，自动继承目标位置的上下文

修改文件的访问类型，使用chcon工具

  • t，指定访问类型
  • -R，递归修改

恢复访问类型， restorecon

  • 恢复为所在位置的默认上下文属性
  • -R，递归修改

调整SELinux布尔值

  • 使用 getsebool 查看
    a，可列出所有布尔值，on 开启， off 关闭
  • 使用 setsebool 设置
    -P，永久更改，重启后仍然有效

查看SELinux日志排查

  • 开启selinux，严格模式
  • 启动httpd服务，修改为非80端口，无法启动，启动报错
  • 查看SELinux日志，排查问题，需要用到setrubleshoot 查看日志，执行下面命令，sealert -l 0f7e3663-d739-4bd1-bbca-c08c652b48cd，查看如何修改 执行DO里的内容，注意里面的变量，PORT_TYPE

总结就是，开启selinux后，想使用非标准接口，那就需要使用selinux命令，允许使用其他的端口；

集群

集群分类

  • 高性能计算集群HPC
    通过以集群开发的并行应用程序，解决复杂的科学问题
  • 负载均衡( LB )集群
    客户端负载在计算机集群中尽可能平均分摊[
  • 高可用( HA)集群
    避免单点故障，当一个系统发生故障时，可以快速迁移
  • 原理
    LVS原理和nginx不同
    lvs原理是路由器，转发数据包，性能高，只能做转发，功能单一；
    nginx原始代理，需要处理下数据包，功能多，比如健康检查；
